## 1. 插件简介

### 1.1 这个插件能做什么？

假设你的项目中有多个 JSON 配置文件，你想找到所有包含 `database.host` 这个键的位置。通常你需要：

1. 打开每个 JSON 文件
2. 手动搜索 "database"
3. 再找到 "host"
4. 记下位置...

**使用 Json Path Navigator**，你只需要：

1. 按下快捷键 `Ctrl+Alt+Shift+F`
2. 输入 `database.host`
3. 立即看到所有匹配结果！

### 1.2 主要功能

| 功能 | 说明 |
|------|------|
| 🔍 快速搜索 | 通过键路径快速定位 JSON 键 |
| 💡 智能建议 | 输入时自动提示可能的路径 |
| 👁️ 预览 | 显示匹配项的值预览 |
| 📍 一键跳转 | 双击结果直接跳转到文件位置 |
| 🖱️ 右键菜单 | 选中文本后右键即可搜索 |

### 1.3 支持的 IDE

- IntelliJ IDEA (Community & Ultimate)
- WebStorm
- PyCharm
- GoLand
- 其他 JetBrains IDE (2024.1+)

## 2. 安装方法

### 方法一：从 IDE 插件市场安装（推荐）

1. 打开 IDE 设置
   - Windows/Linux: `File` → `Settings`
   - macOS: `IntelliJ IDEA` → `Preferences`

2. 进入插件页面
   - 点击左侧 `Plugins`

3. 搜索安装
   - 点击 `Marketplace` 标签
   - 搜索 "Json Path Navigator"
   - 点击 `Install` 按钮

4. 重启 IDE

### 方法二：手动安装

1. 下载插件
   - 从 [GitHub Releases](https://github.com/wenmin92/JsonPathNavigator/releases) 下载最新的 `.zip` 文件

2. 安装插件
   - 打开 IDE 设置 → `Plugins`
   - 点击 ⚙️ 图标 → `Install Plugin from Disk...`
   - 选择下载的 `.zip` 文件

3. 重启 IDE

## 4. 功能详解

### 4.1 搜索对话框

打开搜索对话框后，你会看到：

```
┌─────────────────────────────────────────────────────────────┐
│  Find JSON Key                                        [X]   │
├─────────────────────────────────────────────────────────────┤
│  Search: [database.host_______________] [Search]            │
│                                                             │
│  ┌─────────────────────────────────────────────────────────┐│
│  │ 文件         │ 路径            │ 预览             │ 行号   ││
│  ├─────────────┼─────────────────┼─────────────────┼───────┤│
│  │ app.json    │ database.host   │ host: "local... │  3    ││
│  │ prod.json   │ database.host   │ host: "prod.e.. │  3    ││
│  │             │                 │                 │       ││
│  └─────────────────────────────────────────────────────────┘│
│                                                             │
│  建议:                                                       │
│  ┌─────────────────────────────────────────────────────────┐│
│  │ database.host                                           ││
│  │ database.port                                           ││
│  │ database.name                                           ││
│  └─────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
```

**各部分说明**：

| 区域 | 功能 |
|------|------|
| 搜索框 | 输入要查找的 JSON 键路径 |
| 搜索按钮 | 点击或按 Enter 执行搜索 |
| 结果表格 | 显示所有匹配的结果 |
| 建议列表 | 输入时显示可能的路径建议 |

### 4.2 搜索路径语法

使用 `.` (点) 分隔嵌套的键名：

```
简单键:        name
嵌套路径:      config.database.host
多层嵌套:      app.settings.ui.theme.color
带下划线:      my_config.db_host
```

**示例 JSON 和对应搜索路径**：

```json
{
  "app": {
    "name": "MyApp",
    "settings": {
      "debug": true,
      "log_level": "info"
    }
  },
  "database": {
    "host": "localhost"
  }
}
```

| 搜索路径 | 找到的值 |
|----------|----------|
| `app.name` | "MyApp" |
| `app.settings.debug` | true |
| `app.settings.log_level` | "info" |
| `database.host` | "localhost" |

### 4.3 右键菜单搜索

1. 在代码中选中一段文本（看起来像 JSON 路径的文本）
2. 右键点击
3. 选择 "Find Key in JSON"

```
例如，在你的 Java/Kotlin/JS 代码中：

String host = config.get("database.host");
                         └──────────────┘
                          选中这部分文本，然后右键搜索
```

### 4.4 智能建议

当你开始输入时，插件会自动提供建议：

```
输入: "data"
      ↓
建议: database.host
      database.port
      database.name
      data.users.count
```

点击任意建议即可自动填入并搜索。

## 5. 使用技巧

### 5.1 快捷键

| 操作 | Windows/Linux | macOS |
|------|---------------|-------|
| 打开搜索对话框 | `Ctrl+Alt+Shift+F` | `⌘+⇧+⌥+F` |
| 执行搜索 | `Enter` | `Enter` |
| 关闭对话框 | `Esc` | `Esc` |
| 跳转到结果 | 双击结果行 | 双击结果行 |

### 5.2 高效搜索技巧

**技巧 1**: 从选中文本开始
```
如果你的代码中有路径字符串，先选中它，再打开搜索对话框，
路径会自动填入搜索框。
```

**技巧 2**: 使用建议加速输入
```
只输入前几个字符，然后从建议列表中选择，比完整输入快很多。
```

**技巧 3**: 搜索父路径
```
如果不确定完整路径，可以搜索父路径查看所有子键：
输入 "database" 而不是 "database.host"
```

### 5.3 结果排序

点击表格标题可以排序：
- 按文件名排序
- 按路径排序
- 按行号排序

## 6. 常见问题

### Q1: 搜索没有结果？

**可能原因和解决方法**：

1. **路径拼写错误**
   - 检查键名是否正确
   - 注意大小写（JSON 键是区分大小写的）

2. **路径不完整**
   - 确保使用完整的路径，从根键开始
   - 例如：用 `config.host` 而不是只用 `host`

3. **JSON 文件不在项目中**
   - 确保 JSON 文件在项目目录内
   - 检查文件是否被 .gitignore 排除

4. **文件扩展名不是 .json**
   - 插件只搜索 `.json` 扩展名的文件

### Q2: 如何更改快捷键？

1. 打开设置 → `Keymap`
2. 搜索 "Find Key in JSON"
3. 右键 → `Add Keyboard Shortcut`
4. 输入新的快捷键组合

### Q3: 搜索很慢？

**优化建议**：

1. 输入更具体的路径（减少匹配范围）
2. 关闭不需要的项目文件夹
3. 等待项目索引完成后再搜索

### Q4: 支持 JSON5 或 JSONC 吗？

目前只支持标准 JSON 格式。带注释的 JSON (JSONC) 和 JSON5 暂不支持。

### Q5: 能搜索数组中的元素吗？

目前不支持数组索引搜索（如 `items[0].name`）。只支持对象键的路径搜索。
